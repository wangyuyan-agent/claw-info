# OpenClaw v2026.4.12 版本發佈說明

[GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.12)

## ⚠️ 升級前必讀

### 新功能亮點

- **Active Memory 插件** — 全新可選記憶子代理，在主回覆前執行專屬記憶檢索，支援 message/recent/full context 模式（[#63286](https://github.com/openclaw/openclaw/pull/63286)）

- **Codex 內建 provider** — 含 plugin-owned app-server harness（[#64298](https://github.com/openclaw/openclaw/pull/64298)）
- **安全性強化** — 移除 interpreter-like safe bins、空核准清單防護、shell-wrapper 偵測擴展三項修正

## 功能更新 (Features) - by Star Rating

### ⭐⭐⭐ Active Memory 插件：專屬記憶子代理 ([#63286](https://github.com/openclaw/openclaw/pull/63286))
- **用途**: 在主回覆前啟動一個專屬記憶子代理，自動檢索相關記憶並注入 context，支援 message/recent/full context 三種模式，並可透過 `/verbose` 即時檢視 recall 過程。
- **解決問題**: 過去記憶檢索與主回覆耦合在同一流程中，難以獨立調校或觀察記憶命中情況，導致回覆品質不穩定且除錯困難。
- **影響**: 記憶系統可獨立迭代與監控，使用者能即時驗證 recall 結果，整體回覆品質與可觀察性顯著提升。

```text
┌──────────────────────────┐
│ 使用者送出訊息            │
└──────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ Active Memory sub-agent 啟動  │
│ 依設定選擇 context 模式       │
│ (message / recent / full)     │
└───────────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 執行記憶檢索 (recall)         │
│ /verbose 可即時檢視過程       │
└───────────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 將檢索結果注入主 agent        │
│ context                       │
└───────────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 主 agent 基於豐富 context     │
│ 產生回覆                      │
└───────────────────────────────┘
```

### ⭐⭐⭐ Plugin 載入收斂至 manifest 宣告 ([#65120](https://github.com/openclaw/openclaw/pull/65120), [#65259](https://github.com/openclaw/openclaw/pull/65259), [#65298](https://github.com/openclaw/openclaw/pull/65298), [#65429](https://github.com/openclaw/openclaw/pull/65429), [#65459](https://github.com/openclaw/openclaw/pull/65459))
- **用途**: 將 CLI extension、provider、channel 的啟動範圍收斂至各 plugin manifest 中明確宣告的需求，不再載入未宣告的能力。
- **解決問題**: 先前 plugin 載入會啟動所有可能的 CLI/provider/channel 路徑，造成不必要的記憶體與啟動時間開銷，也增加意外副作用的風險。
- **影響**: 啟動更快、記憶體更省，plugin 行為更可預測，也降低了 plugin 間意外干擾的機會。

```text
┌──────────────────────────┐
│ Plugin 載入啟動           │
└──────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 讀取 plugin manifest          │
│ 解析宣告的 needs              │
│ (CLI / provider / channel)    │
└───────────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 僅啟動 manifest 中宣告的      │
│ 能力，跳過未宣告路徑          │
└───────────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 啟動更快、記憶體更省          │
│ 行為更可預測                  │
└───────────────────────────────┘
```

## 安全性提升 (Security) - by Star Rating

### ⭐⭐⭐ 從 interpreter-like safe bins 移除 busybox/toybox ([#65713](https://github.com/openclaw/openclaw/pull/65713))
- **用途**: 將 busybox 與 toybox 從 sandbox 的 interpreter-like safe binary 清單中移除，防止透過這些多功能工具繞過沙箱限制。
- **解決問題**: busybox/toybox 內建大量子命令（sh、wget、nc 等），若被列為 safe bin，等於開了一扇通往任意指令執行的後門。
- **影響**: 沙箱的指令執行邊界更嚴格，降低透過多功能 binary 逃逸的風險。

```text
┌──────────────────────────┐
│ Sandbox 收到執行請求      │
└──────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 檢查 binary 是否在            │
│ safe bins 清單中              │
└───────────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ busybox / toybox 已從清單     │
│ 移除，不再視為 safe           │
└───────────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 多功能 binary 的執行請求      │
│ 進入正常審核流程              │
└───────────────────────────────┘
```

### ⭐⭐⭐ 防止空核准者清單自動授予核准 ([#65714](https://github.com/openclaw/openclaw/pull/65714))
- **用途**: 當核准者清單為空時，不再將其視為「所有人皆已核准」，而是拒絕核准。
- **解決問題**: 空清單在邏輯上可能被誤判為「無人需要核准 = 已通過」，形成權限繞過漏洞。
- **影響**: 核准流程的 fail-closed 行為更正確，消除了一個潛在的權限提升路徑。

```text
┌──────────────────────────┐
│ 收到需要核准的操作請求    │
└──────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 讀取核准者清單                │
└───────────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ 清單為空？                    │
│ 舊行為：視為全部核准 ✗       │
│ 新行為：拒絕核准 ✓           │
└───────────────────────────────┘
           │
           ▼
┌───────────────────────────────┐
│ Fail-closed：空清單 = 拒絕    │
│ 防止權限繞過                  │
└───────────────────────────────┘
```

### ⭐⭐ 擴展 shell-wrapper 偵測範圍 ([#65717](https://github.com/openclaw/openclaw/pull/65717))
- **用途**: 擴大 shell-wrapper 偵測的覆蓋範圍，識別更多偽裝成一般工具的 shell wrapper。
- **解決問題**: 部分 shell wrapper 未被偵測到，可能被用來繞過沙箱的 shell 執行限制。
- **影響**: 沙箱對 shell 執行的防護更全面。

### ⭐⭐ Gateway auth：清空範例 credential 並拒絕 placeholder 啟動 ([#64586](https://github.com/openclaw/openclaw/pull/64586))
- **用途**: 將出貨範例中的 gateway credential 清空，並在啟動時偵測到 placeholder credential 時直接拒絕啟動。
- **解決問題**: 若使用者忘記替換範例 credential，gateway 會以不安全的預設值運行。
- **影響**: 降低因忘記設定 credential 而暴露 gateway 的風險。
